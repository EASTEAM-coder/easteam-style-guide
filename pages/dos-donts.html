Section 13

Do's
& Don'ts.

The whole guide compressed into pairs. When you're not sure, this is the answer. Print this page. Pin it next to your screen.

Voice & Copy.

Do

  • Lead with the offer or the ask in the first sentence
  • Short sentences and real words
  • Name the set + list what's inside (e.g. Daydream Set — skirt + top)
  • Pick one tone per email: funny / quirky / sassy / bossy
  • One CTA per email, one primary per screen
  • Read it out loud — if you wouldn't text it, don't send it

Don't

  • No mystery numbers (EST 0042)
  • No poetic filler (softer than expected, uncracked, chapter boasts)
  • No vague references (the second piece with no piece named)
  • Don't run all four tone moods at once
  • Don't compete with yourself — never two primary CTAs
  • Don't address her as "ladies / girls / babes"
  • Never use the long dash "—" in any copy, internal or external (use a period, comma, or colon instead)

Design & Color.

Do

  • Black + three pinks only (Light, Dusty, Deep Dusty)
  • Page background always #0A0807 on dark · #F5F0EC on light
  • Light Pink #F9DDE1 — dark theme accent only
  • Deep Dusty #8E4F58 — light theme / email only
  • Dusty Pink #C49099 — one pop per surface
  • Body in Jost Light 300, headers Roboto Mono ALL CAPS
  • Staatliches for wordmark + display only
  • Asymmetric layouts — off-balance is the brand

Don't

  • Don't use Light Pink on Warm Ivory — it disappears
  • Don't use Deep Dusty on black — it goes muddy
  • Don't use any retired pink (#F2D8DB, #D89AA4, #864048, #3A1218, #F2C6CC) on web/app surfaces
  • Exception — email uses Dusty Pink #D9A5AC + Blush #F9EAEC (the notification palette); these are not retired there
  • Don't use italic fonts — anywhere
  • Don't use Roboto Mono for body
  • Don't use Staatliches outside wordmark / display
  • Don't recolor or modify the logo
  • Don't center-align everything — symmetric is off-brand

Buttons & UI.

Do

  • Primary = black bg, pink text, pink border, hover inverts + lifts 2px
  • One primary per screen — max
  • Outline = pink border, fills dusty pink on hover
  • Ghost only for tertiary navigation links
  • Keep button padding tight — 11px vertical on Medium
  • Always 4px radius
  • Always 38px min-height on Medium+ for tap target

Don't

  • Don't put Ghost on an add-to-cart, checkout, or sign-up
  • Don't use two primary buttons on the same screen
  • Don't bloat vertical padding "for breathing room"
  • Don't change the radius
  • Don't drop the border on Primary — the border is the silhouette
  • Don't use Light Pink primary on a Warm Ivory email — switch to Deep Dusty

Email & Social.

Do

  • Body at 12–13px on email + social — never web sizes
  • One reason per email: Sell / Collect / Testimonial
  • Use Deep Dusty #8E4F58 as the email accent
  • Inline styles only in email HTML (Gmail strips <style>)
  • Asymmetric layout in every email
  • Caption line 1 = the headline, the offer, the ask
  • 5/7 rule on IG Story: 5 lifestyle, max 2 catalog
  • Info as design — cute boxes and cards, not text walls
  • Open transactional emails with "Hey [name]."
  • Start hard moments (canceled / declined) with empathy

Don't

  • Don't open with "Hi babe, hope you've been good"
  • Don't use two CTAs in one email
  • Don't write captions longer than 3 lines for feed
  • Don't post catalog stills on Story
  • Don't copy-paste the same caption across IG / TikTok / Pinterest
  • Don't write a 4-paragraph email — tighten or split
  • Don't promise human replies you can't deliver
  • Don't write "recently browsed" into a Shopify notification — that's Klaviyo

Imagery.

Do

  • Catalog: 3:4, 4K, transparent BG, 5 poses per look
  • Lifestyle: candid, on-body, real environments
  • Two-source soft natural lighting
  • Keep lighting + color consistent across a whole drop
  • Edit invisibly — if anyone notices, it's too much

Don't

  • Don't oversaturate or push contrast
  • Don't distort body proportions
  • Don't mix harsh shadows with soft frames
  • Don't ship without final review by Creative Director

Files & Folders.

Do

  • Pattern: Parent_Subcategory_Detail_Number.ext
  • Underscores only between segments
  • CamelCase inside segments (MaiSterling, not mai_sterling)
  • Zero-padded numbers (01, not 1)
  • Every file inherits its parent folder's prefix
  • Date stamps as _MMDDYYYY on folder names
  • Use the two-step temp rename for batches

Don't

  • No spaces, no hyphens, no special characters in filenames
  • No single-digit numbers (use 01, not 1)
  • Don't skip the parent prefix because "it's obvious"
  • Don't batch-rename in one pass — overwrites happen
  • Don't leave duplicates in a sequence — inspect before final numbering
