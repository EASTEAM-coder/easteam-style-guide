Section 09 · The Playbook

Klaviyo
Flow Rules.

The build rules refined in production while redesigning the Welcome Flow. These are additions to email-playbook.html — the things we learned only by shipping into Klaviyo and reading it on a real phone. When a rule here conflicts with an older note, this page wins.

— 01 · Fonts, sized for Klaviyo

Klaviyo does not reliably load custom web fonts across inboxes, so email uses an inbox-safe stack. Two fonts only. No script fonts, no third family, no signature scripts.

RoleFamilyWeightSizeNotes
Headers, eyebrows, labels, buttonsArial500 (Medium)9–26pxNever bold (700) — reads too thick. ALL CAPS, tracked wide.
Body, letters, captionsJost Light, fallback Helvetica30012–13pxLetters may run 13px / 1.8–1.9 line-height for readability.
WordmarkLogo image (PNG)——Header/footer logo files. Never set the wordmark as live text in email.
Type rules — Klaviyo

Arial replaces Roboto Mono as the on-brand header face inside email (same caps, same wide tracking, weight 500 not 700). Body is Jost Light 300 with a Helvetica fallback so it degrades gracefully. Never italic. Never a script signature.

— 02 · The no-dash rule

No dashes. Anywhere. This applies to everything we publish — email, website, social, captions, all copy. No em dashes, no hyphens used as punctuation, and no decorative leading — on eyebrows.

Don't

  • EASTEAM started with a feeling — not a strategy.
  • — A LETTER FROM SCARLETT
  • two-faced

Do

  • EASTEAM started with a feeling.
  • A LETTER FROM SCARLETT · FOUNDER
  • two faced

Use a middot · as the separator, or a comma, colon, or period. Arrows (→) on CTAs are fine — they are arrows, not dashes.

— 03 · Voice & copy

Banned / avoid

  • "Main character" / "main character energy." Dated. Never use it.
  • The word "strategy." It sounds calculated.
  • "Confidence comes from within." We reject the cliché.
  • Casual brag compliments ("you've got great taste").
  • Walls of text. Keep it simple, balance copy with image.

Say instead

  • Confidence you can wear. The right piece changes how you stand, walk in, let yourself be seen.
  • "It started with a feeling."
  • Every version of you is real. You are one woman at home, another with friends, another with strangers. The world calls it two faced. It isn't. Wear them all, never apologize.
  • Classic compliments: "Your taste is exquisite. Let us dress it."
  • Edge is welcome, kept classy: honest, loud, unafraid.
Salutation & sign-off

Style HEY LOVE, as a pink all-caps label (same treatment as an eyebrow, Deep Dusty). Sign off simply: With all my love, then SCARLETT, FOUNDER OF EASTEAM in Arial caps. No handwritten script.

Use {% unsubscribe_link %} (outputs a URL) inside your own styled <a>. Never use {% unsubscribe %} inside an href — it outputs a whole anchor, nests, and leaks the raw style="..." text into the footer. Style the link pink (#D9A5AC), underlined.

— 07 · Image treatments

A
Text on image

Full-bleed photo, text overlaid. Light text with a soft text-shadow if there is no panel behind it.

B
Frosted band

Full-width band across the bottom third. No border, no corner radius (it is a band, not a floating card). Transparent black rgba(0,0,0,0.42) + backdrop-filter: blur, white text. Blur shows in Apple Mail; degrades to a soft tint elsewhere.

C
Whole-image tint

To lower saturation, lay a flat linear-gradient(rgba(0,0,0,0.10–0.20), ...) over the whole image. Keep it light. Just a touch.

D
Contained band

For a centered quote, put the transparent-black panel behind only the text, not the whole image.

E
Exact aspect

When asked, preserve the image's exact ratio (no crop, no shrink): set the cell height to the image ratio at 600px wide.

F
Clear the subject

Place labels and copy where they never cover the subject. If the photo's baked text sits bottom-left, put the label top-right. If the model sits right, put the quote top-left. For cutout PNGs, place them on the Warm Ivory canvas so transparency blends, and use dark text in the empty space.

— 08 · Mobile (non-negotiable)

Do

  • Give every inner section a class that forces width:100% under 480px.
  • Reduce side padding from 56px to ~24px on mobile.
  • Set explicit heights per breakpoint for background-image cells.
  • Stack side-by-side columns; make side images full-width.
  • Tuck overlaid text into a tighter column so it clears the subject.
  • Always preview the phone view before shipping.

Don't

  • Never leave an inner table at a bare width="600" with no override — it overflows and distorts the whole email on a phone.
  • Never set height:auto on a background-image cell — it collapses to nothing.
  • Don't let overlaid text crowd the subject on a narrow screen.

— 11 · Operations · editing the live flow

The sections above are the brand rules. The notes below are the operational rules — how to edit, bake, save, and verify a live Klaviyo flow email without breaking it. The companion skill is klaviyo-email.

Two surfaces, two tools
Standalone template
/email-editor/{TEMPLATE_ID}/edit — editable via Klaviyo MCP (klaviyo_update_email_template).
Live flow message
/flow/message/{MESSAGE_ID}/content/edit — Chrome only. The MCP cannot write flow-message HTML.

A flow message can be created from a template, but once it's edited inside the flow its HTML detaches. Pushing a template update will not change the live flow message.

Save verification (never skip)
  • Click Save, wait ~3 seconds, watch for the green "Your changes have been saved." toast.
  • Re-read the Ace editor (window.ace.edit(el).getValue()) and grep for the change before navigating away.
  • If the editor still has unsaved state, Klaviyo will block navigation with a "Leave site?" dialog — accept only after the toast and re-read.
No CSS rotation in production

transform: rotate(90deg) renders correctly in Apple Mail and the Klaviyo preview, but Gmail mobile, Outlook desktop, Outlook web, and Yahoo strip the transform — content squishes or stands upright. Rotation-locked phones also can't physically tilt to read sideways content. Don't ship live rotated panels. If the creative direction requires a tilt, bake each rotated panel as a static PNG and ship as a flat <img> — the same image renders identically across every client.

Baking rotated / image panels
  • Navigate Chrome directly to the cloudfront source image so it becomes same-origin — canvas reads then work without CORS taint.
  • Render at 4× display dimensions for retina (a 480×380 panel becomes a 1920×1520 canvas).
  • Use the 9-argument drawImage(img, sx, sy, sw, sh, dx, dy, dw, dh) form so the image cover-crops without stretch. The 5-arg form stretches and blurs.
  • For a calendar series with portrait model photos (~0.56 aspect ratio), keep slot width ≤ imgAspect × slotHeight (~270px for a 480px-tall slot) so height stays the constraining dimension. Otherwise one panel's model looks zoomed in next to the others.
  • Trigger the file via Blob: canvas.toBlob → URL.createObjectURL → a[download]. Chrome's save-to-disk flag is unreliable.
  • After the first auto-download in a tab, Chrome shows "site wants to download multiple files". Tell the user to click Allow, or only the first file lands.
Linking baked panels

Wrap every panel image in an anchor so the full image is tappable. Use style="text-decoration:none;border:0;" on both the <a> and the <img> to suppress the underline and link border.

Closing CTA pattern (shared across flows)

Every flow's closing block uses the cream-background pattern with the pink dashed 2-column coupon and the black SHOP button. Don't invent a black-background closing section for a single email — the body wrapper is 600px, so a 100%-wide black inner table only fills 600px on desktop and leaves cream edges around it. That stripe is what makes a one-off black closing look broken. Reuse the shared block.

Icon transparency caveat

If you ship a small icon as a PNG with a transparent background, Klaviyo may flatten it to JPEG on upload — transparent pixels become black and the icon ends up sitting in a black square. Generate icons with a solid background matching the panel (cream #F5F0EC) so even a JPEG flatten blends in.

Flow timing & reordering
  • Day 0 + no Time-delay block between trigger and the email = fires within ~5 minutes of the trigger. Pop-up signup → Welcome Email #1 is effectively instant.
  • Drag-drop reordering of flow cards is fragile under automation — Wait blocks pile up on one side and drop off the other. Ask the user to drag manually (30 seconds), or swap the content (subject + preview + HTML) in place. Functional swap, action positions and day cadence intact.
Brand library URLs

The reliable, public pattern for the Klaviyo brand library is https://d3k81ch9hvuctc.cloudfront.net/company/R2LGSi/images/{UUID}.{ext}. Anything uploaded shows up under that path and can be referenced directly in email HTML.

— 12 · Klaviyo Liquid is Django, not Shopify

Klaviyo's template engine is Django-based. Several familiar Shopify-Liquid tags simply don't exist in Klaviyo and cause the "Email content will not render: Invalid template tag" banner. These are the four that bit us in production.

Don't writeWrite insteadWhy
{% unless cond %}…{% endunless %}{% if not cond %}…{% endif %}unless isn't a registered Django tag — render hard-fails.
{{ a|default:b|default:c }}Single field referenceChained default: with variable arguments breaks the parser.
style="margin:{% if x %}0{% else %}18px{% endif %};"Move the conditional outside the element (a separate spacer row).Liquid embedded inside HTML attribute values is fragile; most common silent-preview-failure cause.
{{ item.image.src }} · {{ item.product.image_url }}{{ item.product.images.0.src }}Those fields don't exist on Checkout Started events; raw event has product.images[] (an array). Django dot-index .0.src gets the first image.
The unsubscribe token is the one Shopify-style tag that works

{% unsubscribe_link %} inside an href is correct (see section 06). Everything else stays Django.

— 14 · Preview & Test verification

The Preview Email dialog has misled a generation of marketers. Defaults that surprise people:

SettingDefaultWhat it actually does
Radio: Show directly in KlaviyoSelectedRenders in the right pane. No email is sent. Easy to misread as "preview sent" — but no inbox delivery happens.
Radio: Send as an emailNot selectedThis is what actually delivers a test to the address you type.
Send To fieldPre-filled with your login emailEdit it before sending.
Apply smart sending and send time optimization (Advanced Options)VariesIf checked AND the recipient got any email in the last hour, the test is silently skipped. Toggle off for tests.
When the toast lies

Klaviyo's "A preview email is on its way" can appear even when no email leaves the queue. Three silent killers:

  • Liquid syntax error in the HTML — render fails server-side, no event logged. Most often {% unless %} or chained |default:.
  • Preview radio was set to "Show directly in Klaviyo" — no real send happened.
  • Per-message smart sending checked AND recipient got mail in last hour — silently skipped.
Verification protocol
  1. Select Send as an email, then expand Advanced Options and uncheck smart sending.
  2. Wait 60 seconds after firing.
  3. Check the recipient's inbox and spam / Promotions tab.
  4. If nothing arrives, look for an "Email content will not render" banner under the message tile in the flow canvas — that's the Liquid error.
  5. If even unrelated flows aren't delivering, escalate to Klaviyo support — it's account-side, not template-side.

— 15 · Hero overlay · the reliable pattern

position:absolute over an <img> works in Gmail and Apple Mail but Outlook strips it, and Klaviyo's preview renderer is inconsistent. The only reliable cross-client text-on-image pattern in email is <td background> with exact cell dimensions matching the image aspect ratio, plus a VML <v:rect> for Outlook.

Step 1 · Fetch the natural image dimensions

Don't guess. Open the cloudfront image URL in a Chrome tab and read it:

const img = document.querySelector('img');
JSON.stringify({ w: img.naturalWidth, h: img.naturalHeight,
                 ratio: (img.naturalHeight/img.naturalWidth).toFixed(4) });

For a 600px-wide email, cell height = 600 × (naturalHeight ÷ naturalWidth). A 1230 × 1606 hero becomes a 600 × 784 cell with zero crop.

Step 2 · Use the td-background pattern
<td background="<IMG_URL>" bgcolor="#F5F0EC" class="es-hero"
    style="background-color:#F5F0EC;
           background-image:url('<IMG_URL>');
           background-position:center top;
           background-repeat:no-repeat;
           background-size:cover;
           height:784px;"
    valign="top">
  <!--[if gte mso 9]>
  <v:rect xmlns:v="urn:schemas-microsoft-com:vml" fill="true" stroke="false"
          style="width:600px;height:784px;">
    <v:fill type="frame" src="<IMG_URL>" color="#F5F0EC"/>
    <v:textbox inset="0,0,0,0">
  <![endif]-->
  <!-- text content table here, valign="top" or "bottom" -->
  <!--[if gte mso 9]></v:textbox></v:rect><![endif]-->
</td>

Mobile @media: override .es-hero { height: <proportional-value> !important; } so the cell stays at the same ratio on phone. For our 1230×1606 hero at ~360px width, that's 470px.

— 16 · Flow setup checklist

Trigger panel
  • Re-entry: "Allow re-entry after a time period" with 30 days. Default is plain "Allow re-entry" which lets a customer enter the flow on every Checkout Started — duplicate sequences in parallel.
  • Profile filter: Placed Order zero times since starting this flow. Auto-cancels remaining emails if the customer completes the purchase mid-flow. In the Trigger panel under Profile filters (separate from Trigger filters).
Each email message tile
  • Subject + preview text match the deck.
  • Sender: EASTEAM / info@easteamny.com. "Use as reply-to" checked.
  • Skip recently emailed profiles (Smart Sending): UNCHECKED. The account-level Smart Sending Period default of 1 hour is unrelated to previews; this per-message toggle is what causes silently-skipped real sends.
  • UTM Tracking checked.
  • Status: Live (not Draft, not Manual) before going live.
Activation footgun

Setting any one email in a draft flow to Live flips the entire flow to Live immediately, with the congrats modal. If other emails are still Draft, those steps silently skip on real triggers. The clean way to activate everything in one shot: top-right Update status → Flow status → Live → Save. That sets every message in the flow to Live at once.

Conflict prevention · two flows on the same trigger

If you're replacing an existing live flow (e.g. an older Social Scout checkout abandonment) with a new EASTEAM version, draft the old flow FIRST, then activate the new one. Otherwise both fire on every Checkout Started event and customers receive duplicate sequences.

To draft an old flow: open it → top-right Update status → Flow status → Draft → Save. The toast confirms "All configured actions in your flow are now set to Draft."

— 18 · Mobile stack order with desktop alternation

When feature blocks alternate (img-left then text-left then img-left on desktop) but ALL should stack image-on-top on mobile, the text-LEFT row needs a trick: put the image cell first in DOM order, then use dir="rtl" on that row's inner table to flip the cells horizontally on desktop. Inner <td>s get dir="ltr" so the text inside doesn't mirror.

<table dir="rtl" role="presentation" width="100%">
  <tr>
    <td class="es-stack es-review-img" dir="ltr" width="240">…image…</td>
    <td class="es-stack es-review-text" dir="ltr">…text…</td>
  </tr>
</table>

Desktop: dir="rtl" flips visual order — image renders on the right even though it's first in DOM. Mobile: .es-stack turns both cells display:block; width:100%; vertical stacking follows DOM order so image lands on top.

Square crop · uniform photos from mismatched sources
<img width="240" height="240" src="…"
     style="display:block;width:240px;height:240px;
            object-fit:cover;object-position:center;border:0;">

/* Mobile media query · responsive square */
.es-review-img img {
  width: 100% !important;
  height: auto !important;
  aspect-ratio: 1 / 1 !important;
  object-fit: cover !important;
  object-position: center !important;
}

— 19 · Shopify discount integration (recovery flows)

When a flow needs a discount code (e.g. COMEBACK10 in Email 3), two steps:

  1. Create via Shopify MCP create-discount — title, code, percentage, customerEligibility:"all_customers", startsAt (now), no endsAt.
  2. Update combinations via GraphQL mutation discountCodeBasicUpdate so the code stacks with any existing codes the customer already has in their cart. Without this, Klaviyo's auto-apply URL either silently no-ops or triggers a "discounts can't combine" prompt — either way conversion dies.
mutation {
  discountCodeBasicUpdate(
    id: "gid://shopify/DiscountCodeNode/<NODE_ID>",
    basicCodeDiscount: {
      combinesWith: {
        orderDiscounts: true
        productDiscounts: true
        shippingDiscounts: true
      }
    }
  ) {
    codeDiscountNode { id }
    userErrors { field message }
  }
}

Discount URL pattern for recovery: https://easteamny.com/discount/{CODE}?redirect=%2Fcheckout sends them straight back to their cart with the code auto-applied.
