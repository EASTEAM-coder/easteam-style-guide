Notification
Emails.

EASTEAM's full set of Shopify notification emails — order confirmation, shipping confirmation, delivered, store credit issued, order refund, return request (received / approved / declined), order canceled, payment error, order edited, order invoice, shipping update, out for local delivery, return shipping instructions, and customer account welcome — built as production Liquid in one brand system.

This page is the canonical reference for how we build, style, and ship them. The whole system is also packaged as the shopify-notification-email skill so a new session can pick up exactly where we left off. Everything below is the rulebook behind that skill.

— 01 · The build workflow

Everything is generated from one Python source of truth. Never hand-edit the generated .liquid files, they get overwritten on every build.

01
Edit the builder

All five emails live in build_emails.py. Shared header, footer, and HEAD CSS sit once at the top; each email body is a MAIN_* block; a TEMPLATES list wires filename, preheader text, UTM campaign, and pixel.

02
Build

Run the builder. It writes the five .liquid files and prints a sanity table that balances every if/endif/for/endfor. Fix any mismatch before continuing.

03
Render & screenshot

Render to HTML with sample order data, then screenshot with Chromium and actually look at it. Zoom into fine details (corners, borders) to confirm pixels, never trust that the CSS "should" work.

04
Deliver

Save the .liquid + HTML preview into email-templates/ and hand over the links. Paste the Liquid into Shopify → Settings → Notifications. Send a real test before going live.

— 03 · Typography

Jost — body copy. Weights 300 (default), 400, 500, 600. Body 13px / line-height 1.6; statements 22–24px.

Arial — uppercase labels, eyebrows, button text. Bold, letter-spaced (0.14em buttons, 0.22–0.30em labels). Arial replaced Roboto Mono in emails because Roboto Mono rendered badly on the owner's Mac. Keep the same caps + letter-spacing treatment.

Staatliches — wordmark only. In email the logo is a hosted PNG, not live text.

Banned in email body: serif fonts, Italiana, Cormorant Garamond. Not brand fonts.

— 04 · Header & footer

— 05 · Voice & copy rules

Cool girl, bougie, confident, direct, warm. Short sentences. Open with "Hey [name]." Use "Hey babe." in the Ambassador card.

Never use: money, cheap, refund (as body voice), raw.

No dashes anywhere (no em or en dash). Use commas or two sentences. Product names from Shopify data may contain dashes; that is data, not our copy.

Sixteen emails, one system. Each is a MAIN_* block in build_emails.py. The first five are the originals; the rest were added to cover the full Shopify notification surface.

Empathy-first intro → kind general reason → keep-it / resell-it suggestion → EASTEAM20 code chip (links to /discount/easteam20) → testimonial-video Ambassador offer. Pink line divider, never a black band.

— 07 · When each email fires

Each notification has a specific Shopify trigger. Two of them have important gotchas for EASTEAM's setup, flagged below.

Order confirmation

Order is placed and payment is accepted.

Shipping confirmation

A fulfillment is created with tracking.

Delivered

Carrier marks the shipment delivered.

Out for local delivery

Shipment is out for delivery (local/last-mile).

Shipping update

Tracking info on an existing fulfillment changes.

Store credit issued

Store credit is added to a customer's account.

Order refund

A refund is processed on the order (EASTEAM = store credit).

Return request received

Customer submits a return request.

Return request approved / declined

You approve or decline that request in admin.

Return shipping instructions ⚠

Fires when a label or tracking is attached to a return. EASTEAM never issues labels, so on its own it may never fire — the self-ship branch is the fallback and mirrors the approved email.

Order canceled

An order is canceled in admin.

Payment error

A payment attempt fails at checkout.

Order edited

Items on an existing order are changed via order editing.

Order invoice

You send an invoice / draft order with an outstanding balance.

Customer account welcome ⚠

Fires only on classic (password) customer accounts, on activation. EASTEAM is on NEW passwordless accounts (login at account.easteamny.com), so this will not fire — use the Klaviyo welcome flow ("Social Scout | Welcome Flow V2", Added-to-List) instead.

— 08 · Email-client survival rules

These are real bugs we hit. Email HTML is not web HTML.

Rounded corners

A global border-collapse:collapse !important reset kills border-radius. On the card table, add inline border-collapse:separate !important; border-spacing:0; next to the radius. Card radius is 25px. Outlook desktop ignores radius entirely, accept square corners there.

Shadows

box-shadow is stripped by Outlook, most Outlook web, Yahoo, and Android Gmail. Don't rely on it. We use a simple 1px Dusty Pink border instead of a shadow.

Layout

No flexbox, float, transform, writing-mode, or margin:auto — these broke the Ambassador panel in Shopify's preview. Table-based, inline styles only.

Paid block

The "Paid with" section only renders when transactions exist. Always include an {% unless paid_shown %} fallback showing total_price so the preview isn't empty.

Images

Use image_url: width: N (notification filter). img_url: '600x' is for themes and renders blank here.

Ampersands

In any href query string, write every & as &amp; or the URL breaks in some clients.

Mobile side padding

Use the .oc_side class for body padding: 30px on mobile (not 18px — that felt too tight). Desktop stays 56px.

Mobile-centered CTAs

Where a button should center on phones, add .btn-mobile-center (sets margin:auto inside the mobile media query only).

No horizontal scroll

Every email must pass a Chromium check asserting document.body.scrollWidth ≤ window.innerWidth at 360 / 375 / 390 / 430px before delivery.

python-liquid preview quirk

In the preview renderer, nil != blank can render as true and print empty labels. Use truthy {% if x %} checks instead. A trailing &amp; in a preview URL is harmless preview-only noise.

— 09 · Klaviyo "leave a review"

The delivered email's review button fires Klaviyo's "Ready to review" event by carrying order and product data to a landing page that hosts the Klaviyo tracking snippet.

https://easteamny.com/pages/leave-a-review
  ?email={{ email | url_encode }}
  &order={{ order_number }}
  &product_id={{ line_items.first.product_id }}
  &product_name={{ line_items.first.title | url_encode }}
  &product_url={{ shop.url }}{{ line_items.first.product.url }}
  &product_image={{ line_items.first.image | image_url: width: 600 | url_encode }}

The landing page must host the Klaviyo _learnq snippet that fires track('Ready to review', …) on load, and a Klaviyo flow must listen for that event.

Limitation: line_items.first sends only the first item. For multi-item orders, give each product in the items list its own review link so every purchased piece is reviewable.

— 11 · Policies baked into the emails

Refunds: store credit only, valid 1 year. Issued 3–5 business days after an approved return arrives and passes review.

Returns: 7-day return window. Customers pay their own return shipping — we never issue a return label to any customer.

Return address: EASTEAM LLC, 20 Broad St, Unit 2801, New York, NY 10005. (Distinct from the footer's corporate address, 100 E 53rd St, New York, NY 10022.)

No false promises: never offer personal replies, DMs back, or human help we won't actually deliver.

Personalization limit: Shopify notification Liquid can't see browse or cart history or product recommendations. "Recently browsed" grids are a Klaviyo job, not a notification email.

OrderlyEmails base: always delete the mis-pasted customs/duties/taxes closing paragraph that ships in the base templates.
