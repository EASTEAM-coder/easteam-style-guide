Section 15 · Roboto System

Word
Documents.

The document-specific type treatment for all professional Word (.docx) deliverables — questionnaires, briefs, agreements, onboarding guides, internal documents. This page intentionally overrides the general brand rule of Roboto Mono titles: Word documents use a unified Roboto family because a single superfamily reads more cohesive and professional in long-form documents than a monospace/sans mix.

Core principle. One type family across the entire hierarchy. The wordmark, every heading, and all body text relate to each other by design. Cohesion comes from family unity, not from decoration.

00 · Scope

Applies to all professional Word (.docx) deliverables — questionnaires, briefs, agreements, onboarding guides, internal documents. Email and web have their own rules (see §07 Type / §09 Email / §14 Website). This standard governs anything that opens in Microsoft Word and ships as a finished document.

Why Word breaks the general type rule

Brand-wide, titles are Roboto Mono and body is Jost. In Word documents, that mix reads as two separate systems on a long page. We unify the entire document under one superfamily — Roboto — so the wordmark, headings, and body all relate visually. This rule is locked for .docx only.

02 · Color

Four colors, each with one job. Body text always stays Page Black for readability. Dusty Pink is the consistent accent throughout the document.

Page Black
#0A0807
Body text, wordmark, dark-band fill
Page background
#FFFDFE
Page color — white with the faintest pink
Dusty Pink (Rose)
#C49099
Accent: all titles, labels, hairline rules, list numbers, header/footer
Light Pink
#F9DDE1
Text on the dark emphasis band only
Accent rule for Word docs

Dusty Pink Rose is the repeating accent — titles, hairlines, numbers, header/footer all use it. This differs from the social/email "pop = one per surface" rule. In documents the rose is the consistent thread; in other surfaces it is reserved for a single moment.

Body copy always stays Page Black. Do not color body text — readability over decoration.

03 · Typography

One family: Roboto. Two cuts do all the text work — Roboto Light for body, Roboto Medium for everything else. The wordmark is the only outside element.

Tracking in Word

Tracking values translate to Word as: Font → Advanced → Spacing → Expanded By [X] pt. The values above are the literal pt amounts to enter.

The wordmark

  • Font: Staatliches, 0% tracking, Page Black
  • Placed as: embedded image (PNG), top-left — never live text. Embedding it as an image means it renders correctly on any machine regardless of installed fonts.
  • Masthead size: ≈ 2.8" wide
  • Running-header size: ≈ 1.0" wide
  • It is the only non-Roboto element in the document.
Italic rule (locked, brand-wide)

Never italic — anywhere, ever. This applies to documents as it does to every other surface.

04 · Layout & components

Top to bottom, here is how a standard EASTEAM Word document is composed.

Masthead (page 1 only)

  • Wordmark image, top-left
  • Eyebrow label below (e.g. "AI CONTENT CREATOR · APPLICATION QUESTIONNAIRE") — Roboto Medium 8.5 pt rose
  • Dusty-pink hairline rule
  • One intro line in body text (Roboto Light 10.5 pt black)

Running header (pages 2+)

  • Small wordmark image, left (~1.0" wide)
  • Document label, right — Roboto Medium 7.5 pt rose
  • Dusty-pink hairline rule beneath

Footer (all pages)

  • Left: "Confidential — EASTEAM LLC. Do not distribute."
  • Right: "Page X of Y"
  • Dusty-pink hairline rule above. Roboto Medium 7.0 pt rose.

Section titles

Roboto Medium caps, rose. Generous breathing room.

  • Spacing: ~16 pt before, ~7.5 pt after
  • Case: ALL CAPS, +2.3 pt tracking

Numbered items

  • Numbers are Roboto Medium 9.5 pt rose
  • Body text is Roboto Light 10.5 pt black
  • ~6.5 pt space after each item

Bullets

  • Marker: em-dash (—) in rose. Never default round dots.

Dark emphasis band

  • One black bar with Light Pink caps
  • At most one per document — reserved for the single most important block (e.g. the task)
  • Type: Roboto Medium 12.5 pt, +2.5 pt tracking, light-pink on black

Hairline rules

  • Thin dusty-pink lines only
  • Use to separate masthead, header, and footer
  • No boxes, no shading blocks — except the one dark band described above

05 · Do's & Don'ts

Do
  • Keep everything in the Roboto family; the wordmark is the only exception
  • Keep body Page Black, accents Dusty Rose, background near-white #FFFDFE
  • Limit the document to one dark band, used for the single key block
  • Left-align every block of text — masthead, body, headings, footer
  • Use em-dash (—) markers in rose for bullets
  • Place the wordmark as an embedded PNG image (top-left)
  • Turn "Different first page" ON so page 1 carries the masthead and pages 2+ carry the running header
Don't
  • Don't center anything — left-aligned only, throughout
  • Don't use italics, anywhere
  • Don't introduce a third text font — Roboto Light + Roboto Medium + the Staatliches wordmark image, nothing else
  • Don't use round bullets — em-dash only
  • Don't add more than one accent color on a surface beyond the rose
  • Don't color body text — body always stays Page Black for readability
  • Don't set the wordmark as live text — always embed as PNG so the document survives fontless machines

File naming convention (EASTEAM standard)

[Name]_[ContentType]_[MMDDYYYY].docx — e.g. AIContentCreator_Questionnaire_05262026.docx. Underscores only, zero-padded two-digit month and day.

Where source files live

Source files live in Production Work File 2026 first, then are copied out to share. Never work off a shortcut — always open the actual file in its production location to avoid two-copies-drift.
